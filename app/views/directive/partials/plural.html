control Object New in v2.0.13

{} otherwise undefined / null

If some object is passed (via scope binding) functionality is appended to that object. This allows you to get "control" of some underlying logic of the marker directive.

The following functions are added:

getPlurals() - Array of all the child model objects associated with a specific directive.

example

$scope.markerControl.getPlurals();

example: get a specific ChildMarker by id/idKey

$scope.markerControl.getPlurals().get(id)

updateModels(modelsArray) - A function to manually update the directives state without a $watch.

newModels(modelsArray) - A function to manually rebuild (from scratch) the directives state without a $watch. Same as doRebuildAll but manually.

clean() - A function to manually destroy the underlying child models which should remove all gObjects from the map.

getManager() - A function to obtain an underlying gObjects manager (if it has one).

hasManager() - A function to check if a manager exist for this plural directive.

managerDraw() - A function to force a underlying manager to draw and update the maps state. (if it has one)
chunk Falsy or Numeric New in v2.0.13

false or 10, '10', '1000' otherwise 20//default milliseconds

This is the attribute for all plural directives to turn off chunking (false) or change the chunk size.
cleanchunk Falsy or Numeric New in v2.0.13

false or 10, '10', '1000' otherwise 20//default milliseconds

This is the attribute for all plural directives to turn off chunking (false) or change the chunk size. For a directives destruction. Therefore this sets chunking on the iterator in scope.$destroy.
doRebuildAll expression boolean property to tell the directive to refresh all the markers on any kind change to models. (Think legacy "flash"). Yes sometimes this is wanted.
