New in v2.0.0

The drawing-manager directive is used to turn on the drawing manager GUI which allow the user to place their own markers, polygons, etc on the map. See the documentation on the Drawing Library.

Note that use of this directive requires the drawing library to be included on the Maps API bootstrap URL, as described in the Drawing Library documentation.

Usage

Parameters

Required properties are in red.

Param Type Details
options expression

The expression should either evaluate to an object containing DrawingManagerOptions for the drawing manager, or to undefined / null .

If it evaluates to an object, by default the object will be $watch()ed, and changes to the options will update the drawing manager. See the static attribute for a way to turn this feature off.
control expression

The expression should either evaluate to an empty object ({}) or undefined / null .

If the result is an object, functionality is appended to that object. This allows you to get "control" of the underlying DrawingManager instance.

The following functions are added:

  • getDrawingManager() - Returns the DrawingManager instance

NOTE: Using getDrawingManager().setOptions() will not update the object passed into the options attribute.
static string

Boolean property which indicates that the options passed in should not be monitored for changes.

This will improve the performance of the directive, but has no other effect.

Changes to the static declaration are ignored after the directive is instantiated. (It is NOT WATCHED.)
