The google-map directive is the core of all other directives. It is used to instantiate a Google Map object in your page. Every other directive must be contained within a google-map for them to work.

Usage

	zoom='{expression}'
	draggable='{string}'
	dragging='{expression}'
	refresh='{expression}'
	options='{expression}'
	events='{expression}'
	bounds='{expression}'>

	...

</google-map>
<google-map center='map.center' 
	zoom='map.zoom'
	draggable='true'
	dragging='{expression}'
	refresh='isMapVisible()'
	options='map.options'
	events='map.events'
	bounds='map.bounds'>

	...

</google-map>
$scope.map = {
	options: {
		mapTypeId: google.maps.MapTypeId.TERRAIN, // default value: ROADMAP
		zoomControl: false // disable zoom controls on map
	},
	center: {
		latitude: 43,
		longitude: -73
	},
	zoom: 8,
	events: {

	},
	bounds: {
		northeast: {
			latitude: 60,
			longitude: -60
		},
		southwest: {
			latitude: 20,
			longitude: -84
		}
	}
};

$scope.isMapVisible = function () {
	return true; // implement your own visibility logic
}

Parameters

Required properties are in red.

Param Type Details
center expression Expression to evaluate as an object representing the map center. The object must have latitude and longitude numeric properties.
zoom expression Expression to evaluate as the map's zoom level. This must be a number between 1[1] and 20[2].
draggable string true if map must be draggable, false otherwise
refresh expression Expression which, if it evaluates to true, will trigger a map refresh. Useful when the map is initially hidden.
options expression Additional options to pass to the google.maps.Map constructor. See all map options
events expression Custom events to apply to the map. This is an associative array, where keys are event names and values are handler functions.
bounds expression Fit the map in the specified bounds. The expression must resolve to an object having both northeast and southwest properties. Each of those properties must have a latitude and a longitude properties.
